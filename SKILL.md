# Pet Management Frontend

Ground every UI decision in the product purpose from `README.md`: help pet owners manage daily care routines clearly, reliably, and pleasantly.

Design for a clean, modern, cute interface that feels warm and welcoming without becoming childish, cluttered, or overly decorative.

Support these core flows clearly in both layout and interaction design:

- Register and manage pet profiles
- Create and complete daily care tasks
- Track what is pending versus completed
- Record mood and behavior notes

Favor distinctive layouts over generic dashboard templates. Create clear hierarchy, strong grouping, and purposeful spacing so the interface feels intentional and easy to scan.

Use soft, lightly colored palettes as a base, but keep the system deliberate:

- Maintain readable contrast for text, controls, and status states
- Use color to clarify task state, mood logging, and navigation
- Add pet-related visual cues only when they improve recognition or warmth

Choose typography that feels friendly and polished. Prioritize readability, hierarchy, and rhythm over novelty.

Use motion with restraint. Add transitions or reveals only when they support orientation, feedback, or delight without slowing down routine task completion.

Keep the implementation production-minded:

- Use semantic HTML structure
- Preserve keyboard accessibility and visible focus states
- Make controls easy to understand and operate
- Ensure layouts work well on mobile and desktop
- Keep components visually consistent across screens

When refining screens or components, improve both aesthetics and usability together. Do not treat styling as separate from task clarity, interaction feedback, or accessibility.

Avoid these anti-patterns:

- Generic AI-looking cards and dashboard grids
- Overused glassmorphism, heavy blur, or trend-driven effects without purpose
- Pastel-on-pastel combinations that reduce legibility
- Decorative pet motifs that compete with core content
- "Cute" styling that weakens structure, hierarchy, or professionalism

Keep solutions lean and context-aware. Do not introduce unnecessary frameworks, boilerplate, or speculative structure that is not supported by the current project.
